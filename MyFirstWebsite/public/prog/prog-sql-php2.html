join/index.php

ブラウザ表示

コード


<form action="" method="post" enctype="multipart/form-data">

    <dl>
        <dt>ニックネーム<span class="required">必須</span></dt>
        <dd><input type="text" name="name" size="35" maxlength="255"></dd>
        <dt>メールアドレス<span class="required">必須</span></dt>
        <dd><input type="text" name="email" size="35" maxlength="255"></dd>
        <dt>パスワード<span class="required">必須</span></dt>
        <dd><input type="password" name="password" size="10" maxlength="20"></dd>
        <dt>写真など</dt>
        <dd><input type="file" name="image" size="35" maxlength=""></dd>
    </dl>
    <div><input type="submit" value="入力内容を確認する"></div>

</form>

postでデータ送信。action=""なのでこのファイル自身にフォームのデータをpostで送る。
写真などを扱うので「enctype="multipart/form-data"」を指定する。 入力してもらうinputは4つ用意。「name」「email」「password」「image」
写真以外は必須を明記。
最後のinputのsubmitで入力してもらった5つ(最低4つ)のデータを送信する。

    join/index.phpからPOST送信するデータ
  • name
  • email
  • password
  • image

1.必要な機能を考える

まずは今回のTwitter風に必要な機能を考えていきます。
このプログラムで〇〇ができるを考えて書き出す！
とにかく思いついたものを書き出して、あとから絞り込んでいく。というやり方が良い気がします。ただし1から考える場合には、機能を考えるにもどうしたら良いかわからないこともあると思いますので、例えば今回の様に何かのプログラムや構造を元にして考えるのも良いかと思います。

  • 〇〇できる
  • 〇〇できる
  • 〇〇できる

↑によって考えたものが↓

  • 投稿
  • 返信
  • 会員登録
  • 退会
  • 写真のアップロード
  • ログイン
  • ログアウト

これらが今回の主要機能となります。

    • □□の情報が必要
  • 〇〇できる

    • □□の情報が必要
  • 〇〇できる
    • □□の情報が必要
    • □□の情報が必要

  • 投稿 (メッセージ、会員情報)

2.で並べた情報に対してグループ分けを考え、テーブルとする。

  • 〇〇できる
    • □□の情報が必要 →投稿グループ
    • □□の情報が必要 →会員グループ
  • 〇〇できる

    • □□の情報が必要 →投稿グループ
    • □□の情報が必要 →投稿グループ

↑により今回は二つのグループに分けられそうなので、↓のテーブルを作ることにする。

  • 会員テーブル(members)
  • 投稿テーブル(posts)

4.情報を整理する

足りていない情報がないか検討し、固有情報、プライマリーキーを用意します。
それぞれの情報に関係性を持たせるため、また識別させるために固有情報が必要なので「ID」として用意します。
また、各テーブルの固有なIDとしてプライマリーキーを設定します。
↓の様に設定しました。

  • membersテーブル
    • 会員ID →プライマリーキー
  • postsテーブル
    • メッセージID →プライマリーキー
    • 投稿者ID
    • 返信者メッセージID

また、必要な情報として↓を追加しました。

  • 入会日 (menbers)
  • 変更日 (menbers)
  • 投稿日 (posts)
  • 変更日 (posts)

5.情報に型を付けてカラムを作る

今までに用意した情報をカラムに置き換えます。
その際に、名前と型を考えて、用意していきます。
名前に関してはローマ字ではなく英語の単語が良い様です。出来るだけ的確に表現した名前を付けていきましょう。

membersテーブル(会員グループ)
カラム名 型 情報 主キー AI
id INT 会員ID 主 AI
name VARCHAR(255) ニックネーム
email VARCHAR(255) メールアドレス
password VARCHAR(100) パスワード
picture VARCHAR(255) 写真のパス
created DATETIME 入会日
modifled TIMESTAMP 変更日
posts(投稿グループ)
カラム名 型 情報 主キー AI
id INT メッセージID 主 AI
message TEXT メッセージ
member_id INT 投稿者のID
reply_post_id INT 返信先のメッセージID
created DATETIME 投稿日
modifled TIMESTAMP 変更日
VARCHAR(M)型はMに0〜255までの数字を指定可能。そのまま最大文字数を表している。
