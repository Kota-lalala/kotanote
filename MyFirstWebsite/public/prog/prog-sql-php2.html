ここでは先ほどのコードを部分的に区切って解説していきます。


<?php

require('../dbconnect.php');

    session_start();

php開始
require()で別ファイルのdbconnect.phpを実行しています。
dbconnect.phpは先ほどコードを載せておいたと思いますが、サーバーへの接続コードを記述してあります
session_start()でセッションをスタートさせます。

if(!empty($_POST)) { }のコード(1)POST値のチェック

この部分では、未入力の場合、passwordが4文字以下の場合、imageがjpgかgif以外だった場合に、$errorに値を保存し、後で利用できる様にしています。
$error = array();の部分は書籍には載っておりません。ですが私がコードを書いて実行してみると、どうやら$errorが未定義なのに評価に使われていてエラーが発生している様なのです。ので、ここでまず定義のみしておくことにしました。
if(!empty($_POST)) { }の部分は$_POSTが空ではなかった場合に実行する。
3回に分けて解説をしていきます。
$_POST['name']が''(空)だった場合に$error['name']にblankを入れています。つまりnameが入力されていなかった場合にblankをする。で、後ほどinputの下に表示する項目を追加し、blankだった場合は・・・という具合に使います。
strlen()は文字数を得るファンクションで、4文字以上のパスワードが満たされているかをここでチェック。falseなら$error['password']にlengthを入れ、後で使用します。
inputでファイルパスを指定したことにより$_FILES['image']['name']にその情報が保存されていますので、$fileNameに取り出しておきます。
次にif(!empty($fileName)) {}の部分で、先ほどの$fileNameが空ではないなら{}内のコードを実行。
substr()でファイル名の後ろから3文字を取り出し、$extに保存しています。
次のif文で$extがjpgまたはgifかどうかを確認し、falseなら$error['image']にtypeを代入しておきます。 


// (2)重複アカウントのチェック
if(empty($error)) {
    $sql = sprintf('SELECT COUNT(*) AS cnt FROM members WHERE email="%s"',
            mysqli_real_escape_string($db, $_POST['email'])
            );
    $record = mysqli_query($db,$sql) or die(mysquli_error($db));
    $table = mysqli_fetch_assoc($record);
    if($table['con'] > 0) {
        $error['email'] = 'duplicate';
    }
}
if(!empty($_POST)) { }のコード(2)重複アカウントのチェック

ここでは、同じemailの値が既に登録されていないかをチェックしています。
if(empty($error)) {}つまり、$errorが空だった場合、つまりエラーがなかった場合のみ実行。
「$sql〜」でsql文を$sqlに代入している。ここの文は入力されたemailの値をWHEREの条件に指定して、その値を持つレコードをカウントした数を、データベースから取り出す指示です。
mysqli_real_escape_string()でサニタイズしてPOSTから呼び出した値を、sprintf()により書式を%Sとしてemailの値に使用し、SELECT文を作っている。
「$record〜」の部分で、$dbに対して$sqlのSQL文を実行し、レコードを$recordに取り出しています。「or die()」はエラーが発生した時用。
「$table〜」の部分で、mysqli_fetch_assoc()を使い$recordから取り出したレコードを$tableに連想配列として保存しています。
「if〜」の部分で、もしそのカウント数が0よりも大きいなら(つまり1個でもあれば)エラーにduplicateを入れる。


    // (3)エラーがなかった(empty()が空だったよ)場合に実行
    if (empty($error)) {
        $image = date('YmdHis') .$_FILES['image']['name'];
        move_uploaded_file($_FILES['image']['tmp_name'], '../member_picture/' .$image);

        $_SESSION['join'] = $_POST;
        $_SESSION['join']['image'] = $image;
        header('Location: check.php');
        exit();
    } else {
        echo 'エラーがあります';
    }
}

こちらも$errorが空(エラーがなかった)の場合に実行する。
$imageにファイルのパスを代入している。
date()を使い、今の「年月日時分秒」を取り出してファイル名の先頭に使っている。
そこに$_FILES['image']['name']を付けて、ファイル名としています。
move_uploaded_file(A,B)を使ってファイルをアップロード。指定したファイルAをサーバーのBへアップロードします
$_SESSION['join'] = $_POSTで、POSTの値をSESSIONに保存しています。
$_SESSION['join']['image'] = $image;で、画像のファイル名をSESSIOに保存しています。
これにより、SESSIONを閉じるまではそのデータが利用可能です。
「header〜」の部分でcheck.phpにページを移動させて、次のexit()で今のindex.phpを閉じています。
else以降の部分は$errorがあった場合に、その旨を表示させています。

5.情報に型を付けてカラムを作る

今までに用意した情報をカラムに置き換えます。
その際に、名前と型を考えて、用意していきます。
名前に関してはローマ字ではなく英語の単語が良い様です。出来るだけ的確に表現した名前を付けていきましょう。

membersテーブル(会員グループ)
カラム名 型 情報 主キー AI
id INT 会員ID 主 AI
name VARCHAR(255) ニックネーム
email VARCHAR(255) メールアドレス
password VARCHAR(100) パスワード
picture VARCHAR(255) 写真のパス
created DATETIME 入会日
modifled TIMESTAMP 変更日
posts(投稿グループ)
カラム名 型 情報 主キー AI
id INT メッセージID 主 AI
message TEXT メッセージ
member_id INT 投稿者のID
reply_post_id INT 返信先のメッセージID
created DATETIME 投稿日
modifled TIMESTAMP 変更日
VARCHAR(M)型はMに0〜255までの数字を指定可能。そのまま最大文字数を表している。
