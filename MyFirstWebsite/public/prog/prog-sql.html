現在SQLを学習中につき、学習した事をただ吐き出すページです。細かい解説は無しでチートシート的に作成していこうと思っています。さっと見られるのが良い所です。こちらは「Collapse」で折りたたみ式にして作ってみます。見やすく出来るかな？

25.まとめ〜一通りの流れを盛り込んでみた

テーブルを作る 


CREATE TABLE carts ( id INT , item_id INT , count INT );

主キーを設定する 


ALTER TABLE `carts` ADD PRIMARY KEY(`id`);

主キーをオートインクリメントに設定する 


ALTER TABLE `carts` CHANGE `id` `id` INT(11) NOT NULL AUTO_INCREMENT;

データを入力する 


INSERT INTO carts SET item_id=1 , count=5 ;
INSERT INTO carts SET item_id=2 , count=3 ;
INSERT INTO carts SET item_id=3 , count=1 ;

リレーションを張る(test_table.idとcarts.item_idに) 


SELECT test_table.name, carts.count FROM test_table, carts WHERE test_table.id=carts.item_id;

グループ化して集計をする 


SELECT item_id , SUM(count) FROM carts GROUP BY item_id;

リレーションを張り、かつグループ化して集計する 


SELECT t.name , SUM(c.count) FROM test_table t, carts c WHERE t.id=c.item_id GROUP BY c.item_id;

外部結合させ、かつグループ化して集計する 


SELECT t.name , SUM(c.count) FROM test_table t LEFT JOIN carts c ON t.id=c.item_id GROUP BY c.item_id;

3つのテーブルをリレーションさせ、かつグループ化して集計する 


SELECT m.name , t.name ,SUM(c.count) AS sales_count_sum FROM makers m, test_table t LEFT JOIN carts c ON t.id=c.item_id WHERE m.id=t.maker_id GROUP BY c.item_id ORDER BY sales_count_sum DESC;

以上です。上記の解説(簡単ではございますが)は他のカードに書いてありますので、ご覧いただけたらと思います。

24.3つのテーブルにリレーションを張る（悲鳴があがるので要注意！ぎゃ〜〜） 

SELECT m.name , t.name ,SUM(c.count) AS sales_count_sum FROM makers m, test_table t LEFT JOIN carts c ON t.id=c.item_id WHERE m.id=t.maker_id GROUP BY c.item_id ORDER BY sales_count_sum DESC;

makers」「test_table」「carts」にリレーションを張って「c.item_id」でグループ化して、「m.name」「t.name」「SUM(c.count) AS sales_count_sum」を「sales_count_sum」の降順で並び替えて表示しています。 


SELECT m.name , t.name ,SUM(c.count) AS sales_count_sum

「m.name」「t.name」「SUM(c.count)」を表示してね。そのときSUM(c.count)は「 AS sales_count_sum」とする事。 


FROM makers m, test_table t LEFT JOIN carts c ON t.id=c.item_id

使用するテーブルの指示。「makers」を使うよ。それと「test_table」をベースとして「carts」を外部結合させてね。 


WHERE m.id=t.maker_id

「makers」と「test_table」を内部結合させてね。 


GROUP BY c.item_id

「c.item_id」でグループ化させてね。 


ORDER BY sales_count_sum DESC;

「sales_count_sum」の値で降順で並び替えするんだよ。

以上！！解るけど・・・・長い。辛い。よく頑張った！笑。そのうち慣れると信じて眠らせておこうzzZZ

23.便利なコマンド集 

DISTINCT item_id FROM carts;

「item_id」だけを表示してね。だけど「重複したデータ」は1度だけの表示にしてね。で抽出。 


SELECT * FROM test_table WHERE price>=50 AND price<150;
SELECT * FROM test_table WHERE price BETWEEN 50 AND 149;

上記の2分は同じ指示。ANDで範囲指定するのと同じ意味で、BETWEENを使った記述も出来る。 


SELECT * FROM test_table WHERE id=1 OR id=3;
SELECT * FROM test_table WHERE id IN (1,3);

上記の2分は同じ指示。ORで複数指定するとの同じ意味で、IN()を使った記述の出来る。 


SELECT * FROM test_table LIMIT 10;

LIMITを付け、抽出件数のリミットを10件までに制限しています。 


SELECT * FROM test_table LIMIT 5,10;

また、この様に指示すると、最初5件は飛ばして6件目から表示を開始、その後10件分までにするという指示。 


SELECT item_id , SUM(count) AS sales_count_sum FROM carts GROUP BY item_id;

抽出結果を表示する際のカラムを「SUM(count)」から「sales_count_sum」に置き換えて表示させる事が出来ます。

22.リレーションを更に〜外部結合させる 

SELECT t.name , SUM(c.count) FROM test_table t LEFT JOIN carts c ON t.id=c.item_id GROUP BY c.item_id;

「test_table」をベースとして「carts」にリレーションを張って、そのテーブルから「item_id」でグループ化した後、「name」と「SUM(c.count)」(集計結果)を表示しています。 


SELECT t.name , SUM(c.count)

ここで「t.name」「SUM(c.count)」の値を表示してね。と依頼。（先程と同じ） 


FROM test_table t, LEFT JOIN carts c

次にこの部分で「test_table」をベースとして「carts」のテーブルも使ってねと依頼。更に「t」「c」と呼ぶから。とも伝えている。 


ON t.id=c.item_id

で、「t.id」「c.item_id」を関連付けてね。と依頼。リレーション 


GROUP BY c.item_id;

最後に「c.item_id」でグループ化した状態で行ってね。と依頼。（先程と同じ） 


SELECT t.name , SUM(c.count) FROM test_table t, carts c WHERE t.id=c.item_id GROUP BY c.item_id;

内部結合の書き方その1 


SELECT t.name , SUM(c.count) FROM test_table t INNER JOIN carts c ON t.id=c.item_id GROUP BY c.item_id;

内部結合の書き方その2 


SELECT t.name , SUM(c.count) FROM test_table t LEFT JOIN carts c ON t.id=c.item_id GROUP BY c.item_id;

外部結合で左側をベースとする書き方 


SELECT t.name , SUM(c.count) FROM carts c RIGHT JOIN test_table t ON t.id=c.item_id GROUP BY c.item_id;

外部結合で右側をベースとする書き方

成る程。なんとなく分かった気はするが、ややこしい！！ですよね^^;

21.更に複雑な集計をする〜グループ化とリレーション 

SELECT t.name , SUM(c.count) FROM test_table t, carts c WHERE t.id=c.item_id GROUP BY c.item_id;

「test_table」と「carts」の二つのテーブルにリレーションを張って、「item_id」でグループ化した後、「name」と「SUM(c.count)」(集計結果)を表示しています。 


SELECT t.name , SUM(c.count)

ここで「t.name」「SUM(c.count)」の値を表示してね。と依頼。 


FROM test_table t, carts c

次にこの部分で「test_table」「carts」のテーブルを使ってねと依頼。更に「t」「c」と呼ぶから。とも伝えている。 


WHERE t.id=c.item_id

で、「t.id」「c.item_id」を関連付けてね。と依頼。リレーション 


GROUP BY c.item_id;

最後に「c.item_id」でグループ化した状態で行ってね。と依頼。

20.複雑な集計をする〜グループ化 

SELECT item_id , SUM(count) FROM carts GROUP BY item_id;

FROMで「carts」からデータを抽出するよ。と伝え、
GROUP BYの後の指示で「item_id」でグループ化したものを計算してね。と伝えています。
ので、結果として「item_id」でグループ化した時の「item_id」「SUM(count)」が表示されています。

18.リレーション 

SELECT * FROM makers, test_table WHERE test_table.id=1 AND makers.id=test_table.maker_id;

ANDの前・・・「test_tableのidが1」を指定
ANDの後・・・上記のmaker_idとmakers.idの一致を指定
つまり、idが1の商品情報と、その生産者の情報を一緒に表示してね。という意味。
次に以下の様にすると 


SELECT * FROM makers, test_table WHERE makers.id = test_table.maker_id;

makers.idとtest_table.maker_idを関連付けて、すべてのデータが表示できる。

14.データを並び替える 

SELECT * FROM test_table ORDER BY id ASC;
SELECT * FROM test_table ORDER BY id;

idを見て昇順で整列させる。ASCは省略可能(デフォルトの為) 


SELECT * FROM test_table ORDER BY id DESC;

idを見て降順で整列させる。 


SELECT * FROM test_table WHERE price<100 ORDER BY id ASC;

この様にWHEREと一緒に使うことができ「絞り込み結果を整列させる」という合わせ技が可能。
これで、priceが100未満に絞り込み、なおかつidの昇順で表示する。となる。

13.データを検索する〜WHEREに複数条件を加える 

SELECT id, name, price FROM test_table WHERE price<100 AND name='じゃがりこ';

100円未満で、なおかつ「じゃがりこ」であるデータを探してくれる。 


SELECT id, name, price FROM test_table WHERE price<100 OR name='じゃがりこ';

100円未満であること、または「じゃがりこ」であること、両方にヒットするデータを探してくれる。 


SELECT id, name, price FROM test_table WHERE (id=1 OR id=2) AND price<100 AND name LIKE '%じゃが%';

複雑！！

  • idが1か2であること
  • priceが100未満であること
  • nameが「じゃが」を含んだ文字であること
の条件をすべて満たしたデータを一瞬で探してくれる。凄い！便利。

作成日：2019.3.25

11.データを検索する〜WHEREをもう少し詳しく 

SELECT id, name, price FROM test_table WHERE id=1;

はすでに紹介済みのもの。idが1のデータを探してくれます。 


SELECT id, name, price FROM test_table WHERE name='じゃがりこ';

の様にTEXTの一致でも検索可能。 


SELECT id, name, price FROM test_table WHERE price<100;

とすればpriceが100未満のデータを全て表示してくれます。便利！ 


SELECT id, name, price FROM test_table WHERE price>100;

となれば当然これもできます。priceが100より大きいデータを表示。 


SELECT id, name, price FROM test_table WHERE price<=100;
SELECT id, name, price FROM test_table WHERE price>=100;

は以上、以下の指定で絞り込みが可能。 


SELECT id, name, price FROM test_table WHERE price<>100;

これは？見慣れませんが、priceが100出ないもの全て。となります。
100より小さくて、100より大きいという意味ですね。

4.SQL文のスペース注意点

キーワードがつながってしまう書き方はNG。 つまり「INSERTINTOtest_table」ではコマンドとテーブル名の堺がわからないのでNG。 しかし「id=1,name='カール',price=99;」は記号で区切りがはっきりしてるので、スペースは関係が無い。 


INSERT INTO test_table SET id  =   2 , name =   'じゃがりこ' , price   =     1249;

の様にスペースを入れても問題なし。(見辛いけど)
でも、 


name='   カール'

だと、入力は出来るが、余計なスペースが値として入ってしまうのでNG。
さらに、 


price = 12  49;

もカラムpriceの値として12と49という指示になってしまうのでNG。こちらはエラーになる。
